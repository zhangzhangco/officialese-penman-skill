# 质量检查表

## 一、结构

- [ ] 首段是否在 2 至 4 句内进入主题
- [ ] 是否说明为什么现在需要写
- [ ] 是否明确提出具体问题
- [ ] 是否区分已有进展和现有不足
- [ ] 是否说明作者、单位或项目的工作基础
- [ ] 是否交代文稿主要内容
- [ ] 是否说明目标读者和实际用途
- [ ] 结尾是否只总结后续工作，没有引入新问题

## 二、事实

- [ ] 政策、标准、机构、项目和数据名称是否真实准确
- [ ] 是否存在没有依据的“显著提升”“全面领先”等判断
- [ ] 是否说明资料时间范围
- [ ] 是否说明数据、案例或调查来源
- [ ] 是否把推测写成了确定事实

## 三、语言

- [ ] 每段是否只有一个主要任务
- [ ] 每段第一句是否能概括本段内容
- [ ] 并列项是否保持在 2 至 4 项
- [ ] 相邻段落是否重复使用同一组连接词
- [ ] 是否存在过长句子
- [ ] 是否存在抽象名词连续堆放
- [ ] 是否存在不必要的口号式表达

## 四、场合适配

- [ ] 专著是否介绍全书结构和目标读者
- [ ] 报告是否说明研究对象、方法和资料范围
- [ ] 申报书是否说明拟解决问题和预期成果
- [ ] 技术方案是否尽快进入实际需求和工程问题
- [ ] 标准解读是否说明与正式标准文本的关系
- [ ] 白皮书是否说明统计口径、样本和时间范围

## 五、交付前检查

- [ ] 是否仍有方括号占位符
- [ ] 标题、单位名称和文稿名称是否一致
- [ ] 章节数量是否与正文一致
- [ ] 人名、职务和日期是否准确
- [ ] 字数是否符合要求
