# 示例输入与输出框架

## 示例一：技术专著序言

### 输入

- 名称：《智能显示系统测试与评价》
- 场合：技术专著
- 读者：科研人员、工程技术人员、标准制定人员
- 内容：显示原理、测试方法、设备、案例、发展趋势
- 篇幅：1800 字

### 推荐结构

1. 显示技术和应用变化
2. 新型显示带来的测试问题
3. 问题涉及设备、方法、评价和应用
4. 国内外研究进展与不足
5. 作者团队的测试和工程基础
6. 本书五章内容
7. 写作原则和目标读者
8. 资料说明与后续工作

## 示例二：项目申报书背景

### 输入

- 项目：影院设备运行状态监测系统
- 目的：申请科研项目
- 问题：设备类型多、故障信息分散、人工排查效率低
- 基础：已有监测原型和两家影院测试记录
- 预期成果：系统原型、测试方法、技术报告

### 推荐结构

1. 影院设备数字化和复杂化趋势
2. 当前运维中的具体问题
3. 现有方法及不足
4. 项目团队基础
5. 项目拟解决的问题
6. 研究内容和预期成果

## 示例三：行业白皮书前言

### 输入

- 主题：城市公共充电设施发展
- 资料：2024—2026 年行业数据、企业访谈、城市案例
- 读者：管理部门、企业、投资机构

### 推荐结构

1. 行业规模与需求变化
2. 建设和运营问题
3. 国内外进展
4. 本白皮书数据来源与方法
5. 主要内容
6. 适用范围与说明

## 示例四：技术方案开篇

### 输入

- 项目：实验室视频测试平台改造
- 问题：设备接口不统一、测试步骤重复、结果难以复现
- 目标：建立统一信号链、自动化测试和结果记录

### 推荐结构

1. 现有平台和实际问题
2. 改造目标
3. 方案总体思路
4. 建设内容与验收依据

技术方案不必大篇幅介绍行业趋势，应尽快进入现状、问题和目标。
