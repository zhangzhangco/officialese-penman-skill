# 正式文稿开篇写作 Skill

## 适用任务

当用户要求撰写、改写或检查以下内容时使用本 Skill：

- 专著序言、前言、编写说明
- 研究报告前言、导言、研究背景
- 项目申报书的背景、必要性与研究基础
- 项目验收报告、工作总结、成果汇编前言
- 行业白皮书、发展报告、专题报告开篇
- 技术方案、建设方案、实施方案的项目背景
- 标准研究报告、标准解读、指南说明
- 教材、培训资料、案例集导言
- 企业解决方案、产品技术报告开篇
- 会议致辞、成果发布词、合作倡议

本 Skill 重点解决四类问题：

1. 文稿结构散乱，段落职责不清。
2. 连词单一，句子之间缺少递进关系。
3. 内容显得空泛，只讲形势，不落到问题和工作。
4. 不同场合套用同一种写法，导致语气、长度和重点不匹配。

## 核心原则

### 1. 先判断场合，再选择结构

不得直接套写。先识别文稿属于哪一种场合，再决定是否保留以下内容：

- 宏观背景
- 现实问题
- 国内外进展
- 作者或单位基础
- 文稿内容结构
- 目标读者
- 资料范围与不足
- 后续工作

### 2. 每段只承担一个主要任务

推荐主线：

**背景变化 → 现实价值 → 问题提出 → 问题拆分 → 已有进展 → 现有不足 → 编写基础 → 文稿内容 → 读者用途 → 说明与展望**

短文可合并相邻环节，但不得打乱因果顺序。

### 3. 先写事实，再写判断

不得仅使用“意义重大”“前景广阔”“成果显著”等抽象评价。每个判断前应尽量给出可核实的依据，例如：

- 技术变化
- 市场需求
- 项目数据
- 工程问题
- 标准变化
- 用户反馈
- 典型案例
- 研究成果

### 4. 连词服务于逻辑

优先使用成组句式构建层次：

- 一方面……另一方面……
- 既……也……
- 既要……也要……
- 不仅……还……
- 从……看……
- 在……基础上……
- 基于上述认识……
- 与此同时……
- 由此可见……
- 面向未来……

同一组句式不宜在相邻段落重复超过两次。

### 5. 保持正式，但避免堆砌

每个并列句建议保留 2 至 4 项。动词应具体，例如：分析、验证、比较、总结、制定、建设、测试、应用。避免连续堆放抽象名词。

## 工作流程

### 第一步：识别文稿类型

从用户信息中判断：

- 文稿名称
- 使用场合
- 作者或发布主体
- 目标读者
- 文稿目的
- 是否需要政策、标准、数据或案例依据
- 预计篇幅
- 是否需要介绍章节结构

信息不足时，优先根据用户已提供的材料完成可写部分，并用方括号保留必要占位符；不要虚构机构、项目、政策、数据或成果。

### 第二步：选择篇幅模式

#### A. 精简型：4 至 5 段，约 600 至 1000 字

适合会议致辞、成果发布词、技术方案开篇、项目简介。

结构：

1. 背景与问题
2. 进展与不足
3. 本单位或本项目基础
4. 文稿或成果内容
5. 目标与展望

#### B. 标准型：7 至 8 段，约 1200 至 2000 字

适合专著前言、研究报告、项目申报、行业专题报告。

结构：

1. 背景变化
2. 现实价值与问题
3. 问题分类
4. 已有进展与不足
5. 编写基础
6. 文稿内容与组织
7. 目标读者与用途
8. 说明与展望

#### C. 完整型：9 至 10 段，约 2000 至 3500 字

适合专著序言、行业白皮书、综合研究报告、成果汇编。

结构采用完整主线，不省略资料范围、研究不足和后续工作。

### 第三步：填充事实要素

使用《references/intake_fields.md》中的信息表。没有事实依据的内容不得写成确定结论。

### 第四步：套用结构，不照抄表面句子

优先读取：

- `templates/universal_full_template.md`：完整通用模板
- `templates/standard_template.md`：标准篇幅模板
- `templates/compact_template.md`：精简模板
- `references/scene_adaptations.md`：不同场合的删改规则
- `references/connectors.md`：连接句库

### 第五步：完成自检

使用 `references/quality_checklist.md`，重点检查：

- 是否明确说明为什么现在要写
- 是否提出具体问题
- 是否说明已有工作仍缺什么
- 是否交代作者或单位的工作基础
- 是否说明本文写什么、给谁用
- 是否存在无依据的夸张表述
- 是否存在一段承担多个主要任务
- 是否存在连续重复的连接句
- 是否仍保留方括号占位符

可运行：

```bash
python scripts/check_placeholders.py <文稿文件.md>
```

## 输出要求

默认直接给出可使用的正文，不先解释写作方法。用户要求模板时，再输出模板和填空说明。

正文应满足：

- 首段在 2 至 4 句内进入主题。
- 每段第一句明确本段任务。
- 重要判断有事实或材料支撑。
- 段落之间存在因果、转折或递进关系。
- 结尾不引入新的核心问题。
- 不编造政策名称、机构成果、数据或项目经历。

## 快速调用指令示例

- “按标准型模板，为这份研究报告写前言，1500 字。”
- “把这段项目背景改成申报书语言，突出需求、问题和研究基础。”
- “为一本技术专著写序言，需要介绍五章内容和目标读者。”
- “检查这篇白皮书前言的结构，删除空话，保留正式语气。”
- “将现有序言压缩成 800 字的成果发布词。”
